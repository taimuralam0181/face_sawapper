Face Swapper Project Viva Notes
===============================

১. প্রোজেক্টটা কী?
আমার প্রোজেক্টটা একটা Django-based face swap web application।
এখানে user ২টা image upload দেয়:
- source image = যেই face নিবো
- target image = যেই image-এর উপর বসাবো

তারপর backend local model/CV pipeline use করে source-এর face target image-এ swap করে result generate করে।


২. কী কী কাজ হয়েছে?
- Django দিয়ে full web application বানানো হয়েছে
- Image upload system করা হয়েছে
- Source and target image validation add করা হয়েছে
- Face swap job database-এ save করা হয়েছে
- Background processing system add করা হয়েছে
- Result image media folder-এ save হয়
- Job status track করা যায়: pending, processing, completed, failed
- API endpoint বানানো হয়েছে
- Recent jobs দেখার option আছে
- Result compare করার UI আছে
- Multiple engine support add করা হয়েছে
- Basic test-ও করা হয়েছে


৩. কী model use করা হয়েছে?
Mainly ২ ধরনের approach use করা হয়েছে:

A. Pretrained Local Model
- InsightFace buffalo_l use করা হয়েছে face detection আর landmark extraction-এর জন্য
- inswapper_128.onnx use করা হয়েছে actual face swap-এর জন্য

B. Classical CV Pipeline
- Face landmark detect
- Face alignment
- Triangulation
- Affine warp
- Color transfer
- Seamless cloning

Fallback হিসেবে OpenCV Haar cascade engine-ও আছে।


৪. Simple ভাবে বলতে হলে:
Sir, আমার project-এ আমি local pretrained InsightFace + InSwapper model use করেছি।
সাথে একটা classical computer vision pipeline-ও রেখেছি যাতে explainable approach দেখানো যায়।


৫. Project workflow
১. User source and target image upload দেয়
২. Form validation হয়
৩. Database-এ একটা job create হয়
৪. Background thread-এ processing start হয়
৫. Selected engine অনুযায়ী face detect হয়
৬. Swap generate হয়
৭. Output image save হয়
৮. Job status completed বা failed হয়
৯. Frontend-এ result show করে


৬. Sir যদি জিজ্ঞেস করে "project-এর main objective কী?"
Answer:
এই project-এর main objective হলো ২টা image-এর মধ্যে automatic face swap করা using local AI/CV pipeline, and এটাকে web application হিসেবে user-friendly ভাবে present করা।


৭. Sir যদি জিজ্ঞেস করে "offline/local বলছো কেন?"
Answer:
কারণ এই system-টা cloud API ছাড়া local machine-এ model file দিয়ে run করে।
মানে internet ছাড়া local environment-এ processing করা যায়, যদি required model file থাকে।


৮. Sir যদি জিজ্ঞেস করে "তুমি কি model train করছো?"
Answer:
না sir, আমি model train করি নাই।
আমি pretrained local model use করেছি।
InsightFace buffalo_l দিয়ে face detect and landmark extract করেছি, আর inswapper_128.onnx দিয়ে actual face swap করেছি।


৯. Sir যদি জিজ্ঞেস করে "classical pipeline কেন রাখছো?"
Answer:
Classical pipeline রাখার কারণ হলো:
- step by step explain করা যায়
- thesis/final project-এর জন্য ভালো explainable approach
- manually alignment, triangulation, blending দেখানো যায়
- pretrained model-এর সাথে comparison করা যায়


১০. Sir যদি জিজ্ঞেস করে "pretrained আর classical-এর difference কী?"
Answer:
Pretrained approach:
- বেশি smart
- better quality দিতে পারে
- ready-made model use করে

Classical approach:
- explainable
- pure computer vision based
- landmark, warp, color transfer, seamless clone use করে
- quality সব সময় pretrained-এর মতো হয় না


১১. Sir যদি জিজ্ঞেস করে "database-এ কী save হয়?"
Answer:
Database-এ mainly এইগুলো save হয়:
- job id
- source image path
- target image path
- output image path
- job status
- provider/engine name
- metadata
- error message
- created time, started time, completed time


১২. Sir যদি জিজ্ঞেস করে "metadata-তে কী থাকতে পারে?"
Answer:
Metadata-তে থাকতে পারে:
- source face box
- target face box
- detector name
- face count
- pose difference
- brightness difference
- validation issues
- pipeline type


১৩. Sir যদি জিজ্ঞেস করে "input validation কী কী করছো?"
Answer:
আমি input validation-এ:
- file extension check করেছি
- max upload size check করেছি
- face detect হইতেছে কিনা check করেছি
- source and target-এ ideally একটা clear face আছে কিনা check করেছি
- pose mismatch বেশি হলে reject করতে পারি
- lighting mismatch and face size mismatch-ও verify করি


১৪. Sir যদি জিজ্ঞেস করে "job status কেন লাগছে?"
Answer:
Job status লাগছে কারণ processing instant নাও হতে পারে।
তাই system বুঝতে পারে কাজটা queue-তে আছে, processing-এ আছে, complete হয়েছে, নাকি fail হয়েছে।


১৫. Sir যদি জিজ্ঞেস করে "background thread use করছো কেন?"
Answer:
যাতে image processing-এর time-এ main request block না হয়।
User upload দেওয়ার পর system background-এ কাজ করতে পারে and frontend-এ status দেখাতে পারে।


১৬. Sir যদি জিজ্ঞেস করে "API কেন বানাইছো?"
Answer:
API বানানো হয়েছে যাতে future-এ mobile app, React frontend, বা অন্য system easily integrate করতে পারে।


১৭. Sir যদি জিজ্ঞেস করে "limitations কী?"
Answer:
Project-এর কিছু limitation আছে:
- source আর target image-এ clear face লাগবে
- একাধিক face থাকলে confusion হতে পারে
- extreme side pose হলে result খারাপ হতে পারে
- lighting mismatch হলে artifacts আসতে পারে
- blur, low resolution, occlusion হলে performance কমতে পারে


১৮. Sir যদি জিজ্ঞেস করে "future improvement কী?"
Answer:
Future-এ আমি:
- GPU support add করতে চাই
- multi-face selection add করতে চাই
- better post-processing করতে চাই
- user authentication add করতে চাই
- video face swap support add করতে চাই
- better quality enhancement add করতে চাই
